# Завдання 4 — AI code review та рефакторинг

## 1. Структурований промпт для AI-code review

> **Контекст:** Нижче — функція на JavaScript (Node.js) з legacy-кодової бази.
> Вона фільтрує масив користувачів і повертає рядки для активних повнолітніх.
>
> **Що перевірити:**
> 1. Code smells та застарілі практики (var, == замість ===, надмірна вкладеність).
> 2. Читабельність і назви змінних/функції.
> 3. Чи можна замінити імперативний цикл на методи масивів.
> 4. Потенційні баги та крайові випадки (порожній масив, відсутні поля).
>
> **Обмеження:** Поведінку функції змінювати НЕ можна — лише стиль і структура.
>
> **Формат відповіді:**
> - спершу марковий список знайдених проблем (проблема → чому це погано);
> - потім відрефакторений код у блоці ```js;
> - короткий підсумок, що саме змінено.
>
> ```js
> function process(d) {
>   var r = [];
>   for (var i = 0; i < d.length; i++) {
>     if (d[i].active == true) {
>       if (d[i].age > 18) {
>         r.push(d[i].name + " - " + d[i].age);
>       }
>     }
>   }
>   return r;
> }
> ```

## 2. Знайдені code smells / проблеми (щонайменше 3)

1. **Неінформативні назви** — `process`, `d`, `r`, `i` нічого не говорять про призначення.
2. **Застарілий `var`** — має блокову проблему hoisting; слід використовувати `const`/`let`.
3. **Нестроге порівняння `==`** — `d[i].active == true` допускає неявні приведення типів; треба `===`.
4. **Надлишкове порівняння з `true`** — `active == true` зайве, достатньо `active`.
5. **Імперативний цикл + ручний `push`** — замість декларативних `filter`/`map`.
6. **Глибока вкладеність `if`** — два вкладені `if` ускладнюють читання; об'єднуються в одну умову.
7. **Конкатенація рядків через `+`** — менш читабельна за template literals.
8. **Відсутність документації/типів** — незрозуміла очікувана структура елементів масиву.

## 3. Відрефакторений код

Див. [refactored.js](refactored.js):

```js
function formatActiveAdults(users) {
  return users
    .filter((user) => user.active === true && user.age > 18)
    .map((user) => `${user.name} - ${user.age}`);
}
```

**Що змінено (поведінка збережена):** змістовні назви, `const`-замість-`var` (тут взагалі без проміжних змінних), строге порівняння, об'єднана умова, `filter`+`map` замість циклу, template literal, доданий JSDoc.
